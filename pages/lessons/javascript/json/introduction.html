JSON staat voor JavaScript Object Notation. Het is een lichtgewicht data-interchange formaat.

Waarom JSON?

JSON is een populair dataformaat voor het uitwisselen van gegevens tussen een server en een webpagina. Het is makkelijk te lezen en te schrijven voor zowel mens als machine. JSON is een tekstformaat dat volledig taalonafhankelijk is, maar het gebruikt conventies die bekend zijn bij programmeurs van de C-familie, inclusief C, C++, C#, Java, JavaScript, Perl, Python en vele anderen.

Mijlpalen van JSON

JSON werd in de vroege jaren 2000 ontwikkeld door Douglas Crockford als een lichtgewicht alternatief voor XML. Het idee ontstond uit de behoefte aan een real-time server-naar-browser communicatieprotocol zonder het gebruik van browserplugins zoals Flash of Java-applets12. Hier zijn enkele belangrijke mijlpalen in de geschiedenis van JSON:

  • 2001: De kernideeën van JSON worden gevormd.
  • 2002: Eerste publieke presentatie van JSON.
  • 2005: Yahoo! begint JSON te gebruiken voor sommige diensten.
  • 2006: De website JSON.org wordt gelanceerd.
  • 2013: ECMA-404 standaard wordt gepubliceerd.
  • 2017: JSON wordt een internetstandaard.

JSON werd snel populair vanwege zijn eenvoud en efficiëntie, vooral in JavaScript-omgevingen. Het wordt nu veel gebruikt in webontwikkeling, vooral voor RESTful API's, configuratiebestanden en gegevensopslag in NoSQL-databases zoals MongoDB2.
